## Role and responsibility

The unit training manager owns sustainment interval tracking. The S-1 chief maintains the unit-wide tickler against the T-R Manual sustainment intervals per MCO P3500.72A<Citation source="MCO P3500.72A" /> paragraph 4002.1.e. The S-1 officer briefs currency state to the commander on a fixed cycle. Each Marine holds individual responsibility for completing scheduled sustainment.

Source documents flow from the T-R Manual sustainment intervals per event, the MCTIMS calendar event history, and the per-Marine training record. Sustainment data feeds CRP per MCO P3500.72A<Citation source="MCO P3500.72A" /> Chapter 5. Outputs include the unit currency report, the per-Marine tickler, the sustainment training schedule, and the delinquent event list.

The currency picture drives CRP. Per MCO P3500.72A<Citation source="MCO P3500.72A" /> paragraph 1001 (sustainment tenet), perishable skills atrophy without periodic demonstration. Per MCO P3500.72A<Citation source="MCO P3500.72A" /> paragraph 3003, currency measures against the sustainment interval. Lapse drops the event from CRP credit per Chapter 5.

## Trigger events

- T-R event sustainment interval approaches expiration (60-day tickler).
- Inbound Marine arrives requiring currency baseline.
- Outbound Marine departs requiring final currency check.
- New T-R Manual release changes sustainment intervals.
- Quarterly currency reconciliation cycle opens.
- Pre-deployment currency validation requested.
- Annual sustainment training schedule release.

## Processing workflow

1. Pull sustainment intervals. For each T-R event in the unit training plan, pull the sustainment interval from the T-R Manual per MCO P3500.72A<Citation source="MCO P3500.72A" /> paragraph 4002.1.e. The unit training manager pulls.
2. Maintain a tickler. Build a per-Marine and per-event tickler tied to the last completion date and the sustainment interval.
3. Notify the chain 60 days before lapse. Brief subordinate leaders on upcoming sustainment requirements.
4. Schedule sustainment training. Build calendar events in MCTIMS per MCO 1553.3C<Citation source="MCO 1553.3C" /> paragraph 4.c.(5).
5. Conduct evaluation or retraining. Run sustainment per the T-R event standard per MCO P3500.72A<Citation source="MCO P3500.72A" /> paragraph 7000.3.
6. Document completion in MCTIMS. Upload completion roster and performance evaluation checklist per MCO 1553.3C<Citation source="MCO 1553.3C" /> paragraph 4.c.(5).
7. Update CRP credit. Sustainment maintains CRP credit for E-coded events per MCO P3500.72A<Citation source="MCO P3500.72A" /> Chapter 5.
8. Track delinquent events. For events past the interval, route to the Training Exceptions workflow (Delinquent T-R Event) per MCO P3500.72A<Citation source="MCO P3500.72A" /> paragraph 4006.3.
9. Brief currency state. The S-1 officer briefs the commander on currency posture and lapses.
10. Reconcile monthly. Cross-check the tickler against MCTIMS event history each month.

## Systems of record and forms

### Systems

- MCTIMS UTM Module. Holds event history per Marine driving currency calculation.
- T-R Manual. Defines sustainment intervals per event.
- Unit currency tickler. Local register tracking upcoming and lapsed events.

### Forms

- Per-Marine Tickler Roster. Local roster of events and intervals per Marine.
- Sustainment Schedule. Local schedule for unit-wide sustainment training.
- Currency Report. Generated in MCTIMS per Marine.

## Common pitfalls

- Sustainment interval tracked from training date, not evaluation date. Per MCO P3500.72A<Citation source="MCO P3500.72A" /> paragraph 3003, currency runs from the last evaluation. Track from evaluation date.
- Lapsed events still counted in CRP. Per MCO P3500.72A<Citation source="MCO P3500.72A" /> paragraph 4002.1.e, lapse drops the event from currency. Drop from CRP credit at the lapse.
- 60-day notice skipped. The chain has no warning of upcoming lapse. Set ticklers at 60 days before interval expiration.
- Delinquent events not routed for prereq re-demonstration. Per MCO P3500.72A<Citation source="MCO P3500.72A" /> paragraph 4006.3, delinquent events require re-demonstration of prereqs first.
- Tickler not reconciled to MCTIMS monthly. Local roster diverges from system of record. Reconcile each month.

## Decision points

- Chained event sustainment credit. Per MCO P3500.72A<Citation source="MCO P3500.72A" /> paragraph 4005.1, completion of an upper-level event provides sustainment credit for chained subordinate events. The unit training manager applies chaining credit when valid.
- Branched event credit. Per MCO P3500.72A<Citation source="MCO P3500.72A" /> paragraph 4005.2, the commander may credit similar events across functional areas (branching). The commander decides per case.
- Re-demonstration of all prereqs versus single event. Per MCO P3500.72A<Citation source="MCO P3500.72A" /> paragraph 4006.3, when prereq re-demonstration is unattainable, the delinquent event is re-demonstrated under the appropriate evaluator. The commander calls the path.
