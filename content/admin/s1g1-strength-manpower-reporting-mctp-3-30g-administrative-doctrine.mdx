## Role and responsibility

S-1 admin specialists apply MCTP 3-30G doctrine across the four admin functions. The S-1 chief frames the unit admin posture against the General Administration (GENA), Operational Administration (OPER), Manpower Administration (MPMN), and Personnel Administration (PERA) functions. The 0170 Manpower Officer aligns the MPMN function to the strength-manpower-reporting mission. The S-1 officer signs by direction on the administrative posture brief.

Source documents flow from MCTP 3-30G Chapter 1 Fundamentals (mission, principles, lines of operation, advocacy), Chapter 2 Functions and Components (the four functions and the three components), the NAVMC 3500.3E T-R event matrix (function-coded events GENA, OPER, MPMN, PERA), and the unit admin task list. Outputs include the function-coded T-R event execution log, the LOO mapping at the unit level, and the administrative posture brief.

The function codes shape MCAAT inspection scope. MCAAT inspects by function code with separate evidence requirements for GENA, OPER, MPMN, and PERA.

## The four administrative functions

MCTP 3-30G Chapter 2 defines four administrative functions. The function codes appear in NAVMC 3500.3E T-R event identifiers and drive the inspection and skill development model.

- General Administration (GENA). Day-to-day administrative tasks supporting the unit, including correspondence, directives, reports, awards, legal admin, postal, and casualty assistance. The bulk of the S-1 clerk and adjutant work runs under GENA.
- Operational Administration (OPER). Administrative support to operations, including deployment, exercise, contingency, and combat operations. Includes personnel readiness inputs, deployment manifests, casualty reporting, and reception of casualties.
- Manpower Administration (MPMN). Strength accounting, T-O and T-E management, TOECR processing, BIC management, MMEA coordination, and personnel readiness data. The 0170 officer runs the MPMN function.
- Personnel Administration (PERA). Individual Marine record administration, including service record book maintenance, MCTFS transactions, MOL administration, awards documentation, fitness reports, and the individual record. The S-1 clerk runs the bulk of PERA at the unit level.

## The lines of operation

MCTP 3-30G Chapter 1 frames the administrative cycle as six lines of operation tied together by support:

1. Requirement Development. Identification of a requirement through formal planning.
2. Requirement Validation and Approval. Authority analysis of the requirement.
3. Requirement Sourcing. Fulfillment of the validated requirement.
4. Requirement Deployment. Delivery of the sourcing solution to the point of employment.
5. Requirement Sustainment. Maintenance of prolonged operations.
6. Requirement Redeployment. Reintegration of the force.

Each LOO applies across military awards, the Marine staffing process, civilian position description management, Marine performance evaluation, military pay and entitlements, and other admin tasks per MCTP 3-30G Table 1-1.

## Processing workflow

1. Frame the unit admin task list. Pull the unit task list from NAVMC 3500.3E and the unit METL. Tag each task with the function code (GENA, OPER, MPMN, PERA).
2. Apply the LOO framework. Map each function-coded task to the LOO cycle (development, validation, sourcing, deployment, sustainment, redeployment).
3. Build the admin posture brief. The S-1 chief frames the unit posture by function and LOO. The brief covers training event status, manning posture, readiness inputs, and individual record state.
4. Coordinate with the operations officer. Operations endorses the OPER function task list against the unit operational tempo.
5. Coordinate with the 0170 officer. The 0170 officer owns the MPMN function task list and the manning roster.
6. Execute by function. The S-1 clerk executes GENA and PERA tasks. The 0170 officer executes MPMN. The adjutant coordinates OPER with operations.
7. Track T-R event execution. Pull the function-coded T-R event report from MCATS. Surface gaps in event execution by function.
8. Brief the commander. The S-1 officer briefs the commander on the administrative posture by function on a fixed cycle.

## Systems of record and forms

### Systems

- MCATS. Marine Corps Automated Tracking System for T-R event execution.
- TFSMS. Master T-O system for the MPMN function structure.
- MCTFS. Master personnel record for PERA function execution.
- DRRS. Defense Readiness Reporting System for OPER function readiness inputs.

### Forms

- NAVMC 3500.3E T-R Event Matrix. Function-coded event matrix.
- Unit Admin Task List. Local matrix tying function code to LOO.
- Administrative Posture Brief. Local brief framing the unit admin posture.

## Common pitfalls

- Function codes not applied at the task level. The unit admin task list runs without function tagging. MCAAT inspection routes by function code. Tag every task at the unit level.
- LOO framework treated as theoretical. The framework drives the admin cycle. Apply the LOO mapping at every recurring task (awards, fitreps, MMEA coordination, readiness reporting).
- Function-coded T-R event execution gapped. The 0170 officer pulls T-R event execution from MCATS by function. Gaps in MPMN function events surface manning posture risk.
- OPER function coordination skipped. The adjutant runs OPER coordination with operations. Skipping the coordination leaves operations without admin support during the exercise or deployment.
- Administrative posture brief omitted. The commander runs without admin posture visibility. Brief on a fixed cycle aligned to the unit operational tempo.

## Decision points

- Function code routing for hybrid tasks. Some tasks (deployment manning, casualty reporting) span OPER and PERA. The S-1 chief tags the primary function and notes the secondary function. The function tag drives the inspection routing.
- Active versus Reserve component admin doctrine. MCTP 3-30G Chapter 2 frames the doctrine for both components. The Reserve I&I staff runs the same four functions with adjusted procedural rules.
